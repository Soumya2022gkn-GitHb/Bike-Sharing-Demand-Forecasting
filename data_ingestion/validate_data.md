# Overview

The `validate_data.py` script is responsible for validating the cleaned Bike Sharing dataset before it enters the feature engineering and machine learning pipeline.

This script ensures that the dataset is:
- complete,
- consistent,
- production-ready,
- and safe for forecasting model training.

The validation pipeline checks:
- dataset existence,
- required columns,
- missing values,
- duplicate rows,
- infinite values,
- target variable integrity,
- numeric feature quality,
- and date consistency.

This script is designed following production-quality software engineering practices to support maintainability, debugging, and automated forecasting workflows.

# Key Functionalities

---

# 1. Dataset Existence Validation

The script checks whether:

```text
cleaned_bike_data.csv
```

exists before starting validation.

If missing, the pipeline stops safely.

---

# 2. Required Column Validation

The script validates the presence of critical forecasting columns.

## Required Columns

| Column | Description |
|---|---|
| instant | Record index |
| dteday | Date |
| season | Seasonal category |
| yr | Year |
| mnth | Month |
| hr | Hour |
| holiday | Holiday indicator |
| weekday | Weekday |
| workingday | Working day flag |
| weathersit | Weather condition |
| temp | Temperature |
| atemp | Feels-like temperature |
| hum | Humidity |
| windspeed | Wind speed |
| cnt | Target bike demand |

---

# 3. Missing Value Analysis

The script detects:
- column-level missing values,
- total missing values,
- and potential data quality issues.

---

# 4. Duplicate Row Detection

The validation pipeline identifies duplicate rows that may affect:
- forecasting accuracy,
- statistical analysis,
- and training consistency.

---

# 5. Infinite Value Analysis

The script checks numeric columns for:
```text
+∞
-∞
```

Infinite values can break:
- ML model training,
- normalization,
- and evaluation metrics.

---

# 6. Target Variable Validation

The target variable:

```text
cnt
```

is validated for:
- existence,
- negative values,
- and integrity.

This ensures reliable forecasting outputs.

---

# 7. Date Validation

The script converts:

```text
dteday
```

into datetime format using:

```python
pd.to_datetime()
```

Invalid dates are detected automatically.

---

# 8. Numeric Feature Validation

The script evaluates numeric columns such as:
- temperature,
- humidity,
- windspeed,
- and hourly demand.

Statistics generated:
- minimum value,
- maximum value,
- mean value.

---

# 9. Dataset Statistical Summary

The validation pipeline generates:
```python
df.describe()
```

to provide:
- distribution insights,
- feature ranges,
- and forecasting readiness checks.
